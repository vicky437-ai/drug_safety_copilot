## Phase 1 — Realistic Synthetic Data Generation (D_RAW)

**What makes it realistic:**
- Case IDs follow FAERS format: `US-PHARMA-YYYY-NNNNNN`
- Drug names from top-50 prescribed medications (metformin, lisinopril, atorvastatin, etc.) with actual clinical doses (e.g., "Metformin 500mg BID", "Lisinopril 10mg QD")
- AE terms use real MedDRA preferred terms (e.g., "Hepatocellular injury", "Torsade de pointes", "Stevens-Johnson syndrome")
- Patient demographics match FDA reporting distributions (age bell curve 45-75, ~55% female)
- Country distribution weighted to US (60%), EU (25%), ROW (15%)
- Reporter types: physician (40%), pharmacist (20%), consumer (30%), other HCP (10%)
- Temporal patterns: onset dates 1-90 days post drug start, realistic reporting lag

## Risk Mitigations
| Risk | Mitigation |
|------|-----------|
| LLM returns invalid JSON in Phase 1 | `TRY_PARSE_JSON` + NULL check; retry logic in procedure |
| AI_EXTRACT format differs | Test single row first; have fallback array format |
| FRESHNESS DMF needs TIMESTAMP | Cast REPORT_DATE to TIMESTAMP if DATE type |
| Cortex Search build delay | Proceed to Phase 6 while index builds |
| Intelligence UI unavailable on trial | Call agent via REST API directly from Streamlit |
